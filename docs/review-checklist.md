# Code Review Checklist

## P0: 架构与耦合

- 是否破坏了既有层次边界？
- 是否新增了跨 feature 依赖？如果有，是否有 ADR 与白名单更新？
- 是否引入循环依赖风险？

## P1: 行为与正确性

- 关键路径行为是否保持兼容？
- 异常与边界条件是否处理完整？
- 回滚路径是否清晰可执行？

## P1: 可测试性

- 是否补齐了新增逻辑的测试（或明确了补测计划）？
- 测试是否覆盖关键分支和错误路径？

## P2: 可维护性

- 命名与职责是否清晰，模块是否过大？
- 是否引入重复逻辑，可否抽取到 shared？
- 文档是否更新（README / ARCHITECTURE / ADR）？

## P2: 技术债跟踪

- 本次引入的技术债是否可量化（范围、影响、处理时间）？
- 是否进入迭代技术债池，并明确 owner？
