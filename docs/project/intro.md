# AIMemoryHub 项目介绍

## 一句话介绍
AIMemoryHub 是一个面向个人与小团队的本地 AI 会话知识库系统：把 ChatGPT、Claude、Cursor、Doubao 等多来源对话统一接入、可检索、可追溯、可复盘，并直接服务于日常研发与知识沉淀。

## 项目定位
在真实工作中，AI 使用记录通常分散在多个平台和终端，导致“用过但找不到、踩过坑会重复、经验难复用”。

本项目的定位是：
- 做你的本地 AI 记忆中枢（Memory Hub）
- 做你的跨模型历史检索层（RAG-ready）
- 做你的 Prompt 质量改进与使用复盘面板（Quality & Review）

## 核心能力
- 多源统一接入：支持 ChatGPT / Claude / Cursor / Doubao / Gemini 等历史会话导入与扫描。
- 本地优先存储：基于 SQLite + FTS5 建索引，兼顾检索性能与可维护性。
- 会话级与消息级管理：支持删除会话、消息标签（如工作/生活/AI）与二次筛选。
- 历史检索与证据回溯：通过 `/api/retrieve` 返回相关会话、片段与来源线索。
- 使用复盘与模式发现：通过 `/api/review` 统计高频主题、重复问题、可沉淀 skill 候选。
- Prompt 规则评分：无需调用大模型即可对 Prompt 进行结构化质量评估。
- Prompt 优化（DSPy + 降级）：优先 DSPy 优化，不可用时自动 fallback，保证始终可产出。
- 面向长会话阅读优化：支持多轮问答锚点跳转、思考过程弱化折叠展示，降低阅读噪音。

## 典型使用场景
- 研发场景：快速找回“上次怎么修的”“某组件改法”“某接口排查路径”。
- Prompt 工程：评估当前 Prompt 质量，拿到可执行的改写建议并形成稳定模板。
- 个人知识管理：把碎片化 AI 对话沉淀为可检索的长期资产。
- 团队协作：统一检索入口，减少重复提问与重复试错。
- 复盘沉淀：定期识别高频问题与重复任务，反推自动化流程与 skill。

## 目标用户
- 高频使用多个 AI 工具的开发者
- 需要长期积累 AI 工作流的产品/运营/内容人员
- 希望做“个人私有 RAG 底座”的独立开发者与小团队

## 差异化价值
- 跨平台整合而非单平台绑定：不依赖某一模型生态。
- 本地可控而非云端黑箱：数据在本地，便于审计与迭代。
- 检索 + 复盘 + 优化一体化：不仅“能找到”，还能“用得更好”。
- 对工程场景友好：接口清晰，可直接接入 OpenClaw 等 Agent 体系。

## 交付形态
- 前端：会话浏览、筛选、标签、时间线阅读、Prompt 评分与优化结果展示
- 后端：扫描导入、检索、复盘、评分、优化 API
- 存储：`kb.sqlite`（主存储）+ 可选 JSON 快照

## 未来演进方向
- 更细粒度的消息结构化（如 tool call、思考链路、代码片段类型）
- 更智能的重复检测与内容去重策略
- 自动化周报/月报与沉淀建议推送
- 团队空间与权限模型（在保持本地优先基础上扩展）
