# GBrain V2 快速使用说明

## 一句话理解

现在默认就只走这套知识引擎。

它会先帮你把原始知识材料整理成更像正式知识的候选，再交给你审核。

可以把它理解成：

`原始材料 -> V2 候选 -> 升格审核 -> 正式知识页`

## 日常怎么用

1. 打开 `升格审核`
2. 先看 `Issue` 和 `Pattern`
3. 再看 `Synthesis`
4. 根据标题、摘要、质量分和 Evidence 数量决定是否升格

## 什么时候去看 “GBrain V2 设置与抽检”

只有这几种情况才需要：

- 升格审核里候选数量不对
- 想确认 V2 数据包有没有正常生成
- 想用一个问题手动抽检 V2 检索结果

如果你只是想处理待升格内容，优先待在 `升格审核`，不用先动设置页。

## 现在最实用的判断标准

优先处理这些条目：

- 标题一眼能看懂
- 摘要不是空的
- 质量分高
- Evidence 不止 1 条，或者虽然只有 1 条但已经非常明确

谨慎处理这些条目：

- 标题像 `Session: ...`、日期、日志标题
- 摘要很弱
- 质量提示很多
- 看起来只是上下文，不像长期知识

## 目前自动省人工的边界

这版已经支持：

- Atom 直接进入默认候选池
- 自动把部分 issue / pattern / synthesis 候选送进升格审核

但还没有完全做到“全自动发布”。

目前更准确的定位是：

- 系统帮你预整理、预筛选
- 你只需要对更少、更像正式知识的候选做最后确认
